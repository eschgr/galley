# mdtool

A lightweight, cross-platform (Windows + macOS) desktop application for **viewing and editing markdown files**, designed around a workflow where files are generated by an external tool (e.g. an LLM) and then reviewed and corrected by hand.

> **Status:** prototype / pre-implementation. The full product specification lives in [`docs/PRD.md`](docs/PRD.md).

## What it does

- **Renders** GitHub Flavored Markdown (GFM) plus LaTeX math, with syntax-highlighted code blocks — aimed at faithfully matching typical LLM markdown output.
- **Live split-view** editing: source editor and rendered preview side by side, with synchronized scrolling and live updates as you type.
- **Auto-saves** edits (debounced), with a manual force-save.
- **Auto-refreshes** when a file changes on disk, with conflict handling that never silently destroys unsaved edits.
- **Tabs** for multiple open documents, organized **per project** across separate windows.
- **Formatting shortcuts** (bold, italic, link dialog, inline code, strikethrough, headings, code block, list indent/outdent) so you don't have to remember markdown syntax.

## Planned tech stack

- **Electron** (single-language TS/JS app shell; bundles its own engine — no separate browser dependency)
- **React** UI
- **CodeMirror 6** editor
- **markdown-it** + GFM plugins + a KaTeX-based math plugin (MathJax fallback) for rendering
- **Node** main process for file IO, watching, and the channel listener

See [`docs/PRD.md`](docs/PRD.md) for the full requirements, architecture, build/distribution plan, licensing notes, and the LLM launcher contract (Appendix A).

## Project status

This repository currently contains the **specification only**. Implementation has not started. The next steps, per the PRD, are:

1. A **rendering spike** (PRD §5.1a / R5) — validate GFM + math + code highlighting against real LLM output before building UI.
2. The **instance/channel model** (PRD §5.3) — the per-project listener and file-delivery mechanism.
3. The editor + preview, then formatting shortcuts and the link dialog.
