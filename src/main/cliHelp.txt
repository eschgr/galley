Galley {{VERSION}} — a local Markdown viewer & editor.

Galley renders and edits Markdown files on your machine. It exists for a workflow
where an LLM (e.g. Claude) GENERATES Markdown and a human REVIEWS and corrects it:
the LLM writes a file to disk and opens it in Galley; the human reads the rendered
view, edits if needed, and Galley auto-saves. Galley also auto-refreshes an open
file when it changes on disk, so when the LLM regenerates that file the visible
tab updates on its own. The file on disk is the shared medium between the two —
you (the LLM) produce files; the human reads and corrects them here.

USAGE
  galley [file ...]                    Open one or more Markdown files, each in a tab.
  galley --project <name> [file ...]   Open files in project <name>'s window,
                                         reusing that window if it is already open.
  galley --project <name> --close <file ...>
                                       Close the tab(s) for those files in the window.
  galley --project <name> --set <file ...>
                                       Make the window show EXACTLY those files
                                         (open the missing, close the rest).
  galley --devtools                    Open developer tools on launch.
  galley --help, -h                    Print this help and exit.

  Paths may be absolute or relative (relative resolves against the current
  directory). Passing several files opens several tabs; the first is focused.

  Galley keeps its own per-project state (window coordination and session) in
  ~/.galley by default; set the GALLEY_HOME environment variable to change it.

  Note: "galley" above stands for however the Galley executable is launched on
  this machine — the installed Galley.exe / Galley.app, or `npm start --` in a
  dev checkout. There is not necessarily a command named "galley" on your PATH.

DRIVING GALLEY FROM AN LLM — the launcher contract
  Galley organizes windows by PROJECT. You do NOT manage windows, probe for a
  running instance, or speak any wire protocol — Galley handles all of that
  itself. To show a file you only ever run ONE command:

      galley --project <name> <absolute_file_path>

  - One project => one window. Pass the SAME --project <name> every time for a
    given project and all its files stay in the same window: the first call opens
    the window, and each later call for that name opens a new tab in it (or
    focuses the tab if that file is already open). Use DIFFERENT names for
    different projects to keep them in separate windows.
  - Choose the project NAME as a short, stable, filesystem-safe token identifying
    the project — a readable slug (e.g. "pack-325") or the first 8 hex characters
    of the SHA-256 of the project's ABSOLUTE ROOT DIRECTORY. Use the SAME name
    every time so the same project always maps to the same window, even in a
    later session. Allowed characters: letters, digits, dot, underscore, hyphen.
    Do NOT key the name on a PID.
  - Always pass ABSOLUTE file paths. Re-opening an already-open file just focuses
    its tab (no duplicate). Open several at once if you like:
        galley --project <name> a.md b.md c.md
  - `galley --project <name>` with no file opens (or reuses) the project's window
    with nothing loaded, ready for later files.
  - You can also MANAGE the tab set, not just add to it. To stop showing a file
    (renamed, merged away, no longer relevant), close its tab:
        galley --project <name> --close <absolute_file_path>
    To make the window show EXACTLY a set of files — opening any missing and
    closing the rest in one step:
        galley --project <name> --set a.md b.md c.md
    Closing a file with unsaved human edits prompts the user first; it is never
    discarded silently.

  That is the whole contract. Galley itself decides whether to open a new window
  or route the file into the project's existing one — and it coordinates through
  ordinary files rather than sockets, so it works the same from a sandboxed shell.

WHAT GALLEY RENDERS
  GitHub Flavored Markdown (tables, task lists, strikethrough, autolinks), LaTeX
  math via KaTeX ($...$, $$...$$, \(...\), \[...\]), and syntax-highlighted
  fenced code. Remote https and inline data: images load; links open in the
  system browser. Aimed at faithfully matching typical LLM Markdown output.

THE APP
  A source editor beside a live preview with synchronized scrolling; tabs for
  multiple documents; Show/Hide Source; formatting shortcuts (bold, italic, link,
  headings, lists). Edits auto-save (debounced); external changes auto-refresh
  with conflict handling that never silently discards unsaved edits. Print and
  Export to PDF live in the File menu.
